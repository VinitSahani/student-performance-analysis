Preprocessing & Transformation Timeline

Step 1: Dataset Ingestion & Encoding Check

Loaded the raw student-mat.csv which uses semicolon (;) as a delimiter. Verified data integrity and confirmed that all 395 rows and 33 variables were read successfully. Running check on missing values revealed 0 missing values, confirming the high quality of the UCI dataset.

Step 2: Isolating Academic Dropouts / Exam Absences

Detected {{ log.absent_final_exam_count }} students with a final grade (G3) of exactly 0. However, further checks showed their first and second period grades (G1 and G2) were positive (averaging between 8 and 12). This indicates these students did not score a true 0, but rather missed the final exam or dropped out of the course.

Critical Analytical Decision: Rather than deleting these records, we created a new column status set to "Absent/Dropout" for these records and "Active" for the rest. This prevents skewing statistical averages and correlation results, while preserving their demographic variables.

Step 3: Feature Engineering & Column Mappings

To prepare the dataset for descriptive analysis, we added several calculated and mapped attributes:

New Column Name Formula / Mapping Logic Reason for Adding
attendance_rate ((180 - absences) / 180) * 100 Converts raw absence counts into intuitive percentage rates.
attendance_category Good (≥90%), Average (75-89%), Poor (<75%) Allows grouping of students into clear segments.
average_grade (G1 + G2 + G3) / 3 Represents the continuous learning trend score.
study_hours 1 → "<2 hours", 2 → "2-5 hours", etc. Replaces numeric categorical codes with readable labels.
mother_education 0 → "None", 4 → "Higher Education", etc. Makes charts and summary tables intuitive to read.
Step 4: Clean Data Export

Exported the final preprocessed dataframe to data/processed/student-cleaned.csv as a comma-separated value file. This processed file is cache-loaded by the Flask application, reducing execution latency.
